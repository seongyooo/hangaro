# 한가로 (HanGaRo) — UI 디자인 프롬프트

---

## 서비스 개요

**한가로(HanGaRo)**는 실시간 관광지 혼잡도 기반 여행 코스 추천 웹/앱 서비스입니다.
"한적한 길로 가다"라는 의미로, 네트워크 트래픽 분산 알고리즘을 관광에 적용해
인파를 피한 최적 여행 경로를 제안합니다.

**타깃 사용자:** 2030 자유여행객, 인파 회피 + 나만의 경험을 원하는 MZ세대
**서비스 유형:** 모바일 반응형 웹 (우선), 앱 확장 고려
**디자인 레퍼런스 톤:** 네이버 지도 + Kakao T의 기능성, Toss의 깔끔함

---

## 디자인 방향

- **심플하고 직관적** — 지도가 주인공, UI는 최소화
- **라이트 모드 기본** / 다크 모드 지원 (시스템 설정 연동)
- **모바일 퍼스트** — 엄지 손가락 닿는 영역에 핵심 인터랙션 배치
- **혼잡도 색상 시스템** 전체에 일관되게 적용:
  - 🟢 초록: 한적 (0~30%)
  - 🟡 노랑: 여유 (30~50%)
  - 🟠 주황: 보통 (50~80%)
  - 🔴 빨강: 혼잡 (80~100%)

## 화면 1 — 메인 지도 (진입 화면)

### 레이아웃
```
┌─────────────────────────┐
│  상단 상태바             │
│  [한가로 로고]  [☀️/🌙]  │  ← 헤더 (로고 + 다크모드 토글)
├─────────────────────────┤
│                         │
│                         │
│      지도 전체화면        │  ← 카카오맵
│   (현재 위치 마커)        │
│   (혼잡도 펄스 노드들)    │
│                         │
│                         │
│                         │
├─────────────────────────┤  ← Bottom Sheet (드래그 가능)
│  ▬ (드래그 핸들)         │
│  이동수단: [도보][대중교통][자동차]  │
│                         │
│  📍 목적지를 입력하세요   │  ← 입력 필드
│  + 경유지 추가           │
│                         │
│  [  경로 찾기  ]         │  ← 메인 CTA 버튼
└─────────────────────────┘
```

### 상세 명세

**헤더**
- 좌측: 한가로 로고 (텍스트 or 심볼 + 텍스트)
- 우측: 다크모드 토글 아이콘
- 배경 투명 or 반투명 블러 처리 (지도가 보이도록)

**지도 위 혼잡도 노드**
- 관광지마다 원형 펄스 애니메이션
- 혼잡도에 따라 색상 + 펄스 속도 다름 (혼잡할수록 빠르고 강하게)
- 노드 탭 시 장소명 + 혼잡도 라벨 툴팁 표시
- 현재 위치: 파란색 점 + 외곽 파동

**Bottom Sheet**
- 기본: 드래그 핸들만 살짝 보이는 상태 (peek 상태)
- 위로 드래그: 절반 높이 → 전체 높이 순으로 스냅
- 배경: 흰색 (라이트) / 다크그레이 (다크), 상단 모서리 라운드
- 그림자 위로 올라오는 느낌

**이동 수단 선택**
- 세그먼트 탭: [🚶 도보] [🚌 대중교통] [🚗 자동차]
- 선택된 탭에 primary 색상 강조

**입력 필드**
- 목적지 필드: 📍 아이콘 + placeholder "목적지를 입력하세요"
- 경유지 필드: 🔵 아이콘 + 드래그로 순서 변경 가능
- `+ 경유지 추가` 버튼: 텍스트 버튼 (아이콘 포함)
- 각 필드 우측에 ✕ 삭제 버튼
- 자동완성 드롭다운: 장소명 + 혼잡도 색상 점 함께 표시

**목적지 미입력 시**
- `경로 찾기` 버튼 대신 `주변 한적한 곳 추천` 버튼으로 변경

## 화면 2 — 경로 탐색 중 (로딩)

### 레이아웃
```
┌─────────────────────────┐
│      지도 전체화면        │
│                         │
│  노드들 순차 활성화 →    │  ← Dijkstra 탐색 애니메이션
│  → 경로 초록으로 확정    │
│  → 파티클 흐름 시작      │
│                         │
├─────────────────────────┤
│  혼잡도 분석 중...       │
│  ████████░░ 75%         │  ← 프로그레스 바
└─────────────────────────┘
```

### 상세 명세
- 지도 위에서 탐색 애니메이션이 직접 진행
- 최소 2초 이상 애니메이션 보장 (API가 빨라도 기다림)
- 하단 텍스트는 단계별로 변경:
  1. "관광지 데이터 수집 중..."
  2. "혼잡도 분석 중..."
  3. "최적 경로 계산 중..."
- 취소 버튼 우상단에 작게 배치

## 화면 3 — 결과 화면

### 레이아웃
```
┌─────────────────────────┐
│  ← [한가로]  [저장][공유]│  ← 헤더
├─────────────────────────┤
│  [플랜A 혼잡최소][B 최단][C 숨은명소]  │  ← 플랜 탭
├─────────────────────────┤
│                         │
│   지도 (경로 애니메이션)  │  ← 스와이프로 높이 조절
│                         │
├──────── 스와이프 핸들 ───┤
│  혼잡도 -61% ↓  평균 0.28│  ← 요약 배지
│                         │
│  ⚠️ 경복궁 혼잡 안내카드 │  ← 버퍼 라우팅 제안 (해당시)
│                         │
│  ① 창덕궁 후원  🟢 한적  │
│     도보 15분 →          │
│  ② 낙산공원    🟢 한적  │
│     버스 10분 →          │
│  ③ 이화동      🟡 여유  │
│  ...                    │
│                         │
│  [  다시 검색  ]         │
└─────────────────────────┘
```

### 상세 명세

**플랜 탭**
- A / B / C 3개 탭
- 각 탭에 핵심 지표 한 줄 표시:
  - A: "혼잡도 최소 · 평균 0.28"
  - B: "이동 30분 · 최단 거리"
  - C: "숨은 명소 3곳 포함"
- 선택된 탭의 지도 경로 색상 변경

**스와이프 비율 조절**
- 지도 : 리스트 = 50:50 기본
- 위로 스와이프 → 지도 최소화 (리스트 풀스크린)
- 아래로 스와이프 → 지도 풀스크린 (리스트 숨김)

**버퍼 라우팅 경고 카드**
- 노란 배경 카드 (경고 톤)
- "경복궁은 지금 매우 붐빕니다. 서촌 → 통의동을 먼저 들른 뒤 방문하는 코스를 추천합니다."
- `이 코스로 변경` (primary 버튼) / `원래대로` (텍스트 버튼)

**경유지 카드**
- 번호 원 → 장소명 → 혼잡도 라벨 (색상 뱃지)
- 두 번째 줄: 예상 체류 시간
- 카드 사이: 이동 수단 아이콘 + 소요 시간 (연결선)
- 우측 드래그 핸들 (≡) 으로 순서 변경
- 순서 변경 시 혼잡도 실시간 재계산 → 뱃지 색상 업데이트

**저장 / 공유**
- 헤더 우측 아이콘 버튼
- 공유: 링크 복사 or 카카오톡 공유

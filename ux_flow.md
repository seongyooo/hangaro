# 한가로 UX Flow

> 사용자 경험 흐름 정의 문서

---

## 전체 흐름 요약

```
앱 실행
  └─▶ [화면 1] 메인 지도
          현재 위치 + 주변 혼잡도 노드 표시
          하단 Bottom Sheet — 목적지 / 경유지 입력 + 이동수단 선택
          └─▶ 경로 찾기 버튼
                  └─▶ [화면 2] 탐색 애니메이션 (최소 시간 보장)
                          └─▶ [화면 3] 결과 화면
                                  플랜 A / B / C + 혼잡도 + 드래그 순서 변경
```

---

## 화면 1. 메인 지도 (진입 화면)

### 지도 영역
- 현재 위치 자동 감지 → 지도 중앙 표시
- 주변 관광지 혼잡도 펄스 노드 자동 표시
- 다크 모드 기본값은 라이트 / 시스템 설정 연동도 지원

### 하단 Bottom Sheet
- 네이버 지도 스타일: 기본은 살짝 올라온 상태, 위로 드래그하면 전체 노출
- **이동 수단 선택** (도보 / 대중교통 / 자동차) — 입력창 상단에 탭 형태로 먼저 선택
- **목적지 입력 필드** 1개 기본 노출
- `+ 경유지 추가` 버튼 → 경유지 무제한 추가 가능
- 각 필드 타이핑 시 자동완성 드롭다운
- **날짜/시간 입력 없음** — 실시간 현재 기준으로만 동작
- `경로 찾기` 버튼

### 목적지 없이도 사용 가능
- 목적지 미입력 시 "지금 주변 한적한 곳 추천" 모드로 동작
- 현재 위치 기반으로 혼잡도 낮은 장소를 자동 제안

---

## 화면 2. 경로 탐색 중

### 구성 요소
- 지도 위에서 Dijkstra 탐색 시각화 애니메이션
  - 노드들이 탐색 순서대로 순차 활성화
  - 혼잡 노드 빨간 기둥 진동 후 우회
  - 최적 경로 링크가 초록으로 확정되며 파티클 흐름 시작
- 하단 텍스트: "혼잡도 분석 중..."
- **최소 애니메이션 시간 보장** (API 응답이 빨라도 자연스럽게 유지)

### 목적지 혼잡 안내
- 버퍼 라우팅 제안은 결과 화면에서 표시 (이 화면에서는 표시 안 함)

---

## 화면 3. 결과 화면

### 레이아웃
- 상단: 지도 (노드 + 링크 애니메이션)
- 하단: 경유지 리스트
- 상단/하단 비율을 스와이프로 자유롭게 조절 가능

### 상단 요약
- "혼잡도 OO% 절감" 배지
- "평균 혼잡도 0.XX" 배지

### 플랜 A / B / C
- 목적지 설정 시 3가지 경로 방안을 탭으로 제시
- 탭 전환 시 지도 경로도 함께 변경

| 플랜 | 최적화 기준 | 설명 |
|------|------------|------|
| **A — 혼잡 최소** | 혼잡도 가중치 최대 | 인파를 최대한 피한 경로 |
| **B — 거리 최단** | 이동 거리/시간 최소 | 가장 빠르게 이동하는 경로 |
| **C — 숨은 명소** | hidden_gem_score 최대 | 잘 알려지지 않은 장소 위주 경로 |

### 경유지 리스트
- 각 카드 구성:
  - 장소명 + 혼잡도 라벨 (한적 / 여유 / 보통 / 혼잡)
  - 예상 체류 시간 (소요 시간)
  - 다음 장소까지 이동 수단 + 소요 시간
- **드래그 앤 드롭으로 순서 변경 가능**
- 순서 변경 시 혼잡도 재계산

### 목적지 혼잡 시 버퍼 라우팅 안내
- 상단 경고 카드 표시:
  `"OO은 지금 매우 붐빕니다. 근처 OO을 먼저 들른 뒤 방문하는 코스를 추천합니다."`
- `이 코스로 변경` / `원래대로` 선택

### 하단 액션
- `저장` / `공유` 버튼
- `다시 검색` 버튼

## 확정 사항 요약

| 항목 | 결정 |
|------|------|
| 경유지 수 제한 | 무제한 |
| Bottom Sheet 기본 상태 | 살짝 올라온 상태, 드래그로 전체 노출 |
| 목적지 없이 사용 | 가능 (주변 한적한 곳 추천 모드) |
| 결과 플랜 수 | A / B / C 3가지 |
| 날짜/시간 입력 | 없음 (실시간 기준) |
| 로딩 최소 시간 | 보장 |
| 버퍼 라우팅 팝업 위치 | 결과 화면 |
| 경유지 순서 변경 | 드래그 앤 드롭 |
| 코스 저장/공유 | 결과 화면에 포함 |
| 지도/리스트 비율 | 스와이프로 조절 |
| 이동 수단 선택 위치 | 입력창 상단 (처음에 선택) |
| 로그인 | 없음 (로그인 없이 전체 사용 가능) |
| 다크 모드 | 라이트 기본, 시스템 설정 연동 지원 |

## 미결 사항

- [ ] 여행 스타일 필터 (문화 / 자연 / 음식) 노출 여부 및 위치
- [ ] 플랜 A/B/C 차별화 기준 (혼잡도 최소 / 거리 최단 / 숨은 명소 위주 등)
- [ ] "다시 검색" 버튼 위치 (상단 헤더 vs 리스트 하단)
- [ ] 드래그 순서 변경 후 혼잡도 재계산 시 로딩 표시 방법
